\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Approch and Assumptions\+:-\/}{Approch and Assumptions:-}}\label{index_autotoc_md4}

\begin{DoxyItemize}
\item The robot can only explore in rectangular workspace \mbox{[}(-\/1,-\/1); (-\/1,1); (1,1); (1,-\/1)\mbox{]}. Other space is Non-\/\+Traversable.
\item The robot exploration start at (-\/1,-\/1) and ends at (1,1)
\item If robot unable to reach at end(1,1), then robot fail to explore the workspace.
\item I take a step size of 0.\+1 in both x and y directions.
\item By considering above point make a grid of 21 Rows and 21 Columns.
\item Robot can move in 8 direction (North, South, East, West, North-\/\+East, North-\/\+West, South-\/\+East, South-\/\+West)
\item Done transformation from old co-\/ordinated system (User-\/facing) to new co-\/ordinates system (for calculation and algorithm implementation)
\item Use A$\ast$ search to explore (-\/1,-\/1) to (1,1)
\item User can change obstacle (center co-\/ordinates and diameter) in main.\+py file. (take input in old co-\/ordinate system only)
\item Console prints of obstacle and Path details. (In old co-\/ordinate system)
\end{DoxyItemize}
